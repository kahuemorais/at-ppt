# Contributing

## Getting Started

1. Clone the repository
2. Open `index.html` in your browser
3. No build step required

## Making Changes

1. Create a branch for your feature (`git checkout -b feature/amazing`)
2. Make your changes
3. Test locally by opening `index.html`
4. Commit your changes
5. Push and create a PR

## Guidelines

- Keep CSS organized following the existing structure
- Use CSS variables for colors and spacing
- Test responsiveness at 768px and 1024px breakpoints
- Maintain bilingual support when adding new text
- Update CHANGELOG.md when adding features

## Code Style

- 2 spaces indentation
- CSS variables for theme values
- Semantic HTML elements
- BEM-style class naming
